1. PHP Form Handling

We can create and use forms in PHP. To get form data, we use PHP super global variables $_GET and $_POST. The form request may be GET or POST.

a. PHP GET Form

GET request is the default form request. Data passed through GET is visible in the URL, so it is not secure. Only limited data can be sent. 

File: form1.html

<form action="welcome.php" method="get">
Name: <input type="text" name="name"/>
<input type="submit" value="visit"/>
</form>

File: welcome.php

<?php
$name=$_GET["name"];
echo "Welcome, $name";
?>

b. PHP POST Form

POST request is widely used to submit large data like file upload, login form, registration form etc. Data is not visible in the URL, so it is more secure. 

File: form1.html

<html>
<head> <title> Post Method </title></head>
<body>

<form action="login.php" method="post">
<table>
<tr><td>Name:</td><td><input type="text" name="name"/></td></tr>
<tr><td>Password:</td><td><input type="password" name="password"/></td></tr>
<tr><td colspan="2"><input type="submit" value="login"/></td></tr>
</table>

</form>
</body>
</html>

File: login.php

<?php
$name=$_POST["name"];
$password=$_POST["password"];
echo "Welcome: $name, your password is: $password";
?>

Difference Between GET and POST Method

S.N. GET Method S.N. POST Method
1 Limited data (sent in header) 1 Large data (sent in body)
2 Not secure (data visible in URL) 2 Secure (data not visible in URL)
3 Used for fetching information 3 Used for updating data
4 Can be cached 4 Cannot be cached
5 Default method 5 Must be specified
